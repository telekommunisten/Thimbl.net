Get finger

To be able to use Thimbl, you need to install Finger (at least, if you're) going to run it on your own server.

OK, so, what is finger?

From Wikipedia:

The Finger protocol and the Finger user information protocol are simple network protocols for the exchange of human-oriented status and user information.


Got that? No? OK, the easy version is that Finger is something which allows computers to pass small bits of status information back and forward across a network (in our case, the internet).

Finger was first written in the 1970s and has been largely ignored until we came along and wrote Thimbl (see, a Thimble, fits on a Finger, geddit?)

Right, so, what's xinetd?

From Wikipedia (again):

... xinetd, the eXtended InterNET Daemon, is an open-source super-server daemon which runs on many Unix-like systems and manages Internet-based connectivity. It offers a more secure extension to or version of inetd, the Internet daemon.


Essentially, Finger will connect to the internet using TCP wrappers it finds in xinetd.

OK, that's enough, how do I install it and get it all ready for Thimbl?

  1. Install the xinetd package using your package manager
  2. Install the Finger package as above
  3. Edit your xinetd configuration to enable Finger
  4. Make your users' home directories allow the Finger user to open them (i.e. chmod a+x)
  5. Make sure your .plan file allows the Finger user to read it (i.e. chomd a+r)

OK, OK! If you really want to start playing with Thimbl, you'll need to install Finger on you server (which is pretty easy, see how here) and have your own domain. Got that? Right. Now you can start using thimbl (as long as you have a .plan file structured correctly.)

You don't really want to be using Finger manually, or creating your own .plan files, which is handy, as one of our contributors, Mark Carter, has put together a CLI client (that's a command line client for those who aren't nerds!).

Thimbl is a set of client-side tools for a distributed open source microblogging platform. Thimbl-CLI is a set of command-line tools, and is more modest in its ambitions than the whole of the Thimbl project. - Mark Carter

 

Using Thimbl CLI

Thimbl-CLI requires Linux/cygwin. It also includes the simplest possible finger daemon to serve your messages.

Get the CLI client here by visiting Marks Blog or grabbing the files from Github.

Thimbl-CLI is pretty much ready to go out of the box client-side.

1. Set up your profile:
thimbl setup mcarter@markcarter.me.uk "Just a guy" \ "Mark Carter" "www.markcarter.me.uk" "N/A" "info@markcarter.me.uk"
2. Create a post:
thimbl post "First post"
3. Follow someone:
thimbl follow "dk" "dk@telekommunisten.org"
4. Fetch messages of people you are following:
thimbl fetch
5. Print messages (including your own):
thimbl print
